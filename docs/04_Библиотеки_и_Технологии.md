# Библиотеки и технологии

## Используемые библиотеки

### AndroidX AppCompat (1.6.1)

**Зависимость:**
```gradle
implementation("androidx.appcompat:appcompat:1.6.1")
```

**Назначение:**
- Обеспечивает обратную совместимость с более старыми версиями Android
- Предоставляет класс `AppCompatActivity` - базовый класс для всех Activity
- Поддерживает Material Design компоненты на старых версиях Android

**Почему выбрана:**
- Стандартная библиотека для Android разработки
- Обеспечивает единообразный внешний вид на всех версиях Android
- Необходима для использования Material Design компонентов

**Использование в проекте:**
- Все Activity классы наследуются от `AppCompatActivity`

---

### Google Material Design Components (1.9.0)

**Зависимость:**
```gradle
implementation("com.google.android.material:material:1.9.0")
```

**Назначение:**
- Предоставляет компоненты Material Design
- Включает стили, темы, виджеты в стиле Material Design
- Обеспечивает современный и привлекательный UI

**Почему выбрана:**
- Официальная библиотека Google для Material Design
- Обеспечивает единообразный дизайн с другими Android приложениями
- Включает множество готовых компонентов (кнопки, карточки, и т.д.)
- Поддерживает темную тему

**Использование в проекте:**
- Используется для стилизации кнопок, карточек, полей ввода
- Применяется в темах приложения (themes.xml)

---

### AndroidX ConstraintLayout (2.1.4)

**Зависимость:**
```gradle
implementation("androidx.constraintlayout:constraintlayout:2.1.4")
```

**Назначение:**
- Мощный layout manager для создания сложных макетов
- Позволяет создавать плоские иерархии view (меньше вложенности)
- Обеспечивает гибкое позиционирование элементов

**Почему выбрана:**
- Рекомендуемый Google layout manager
- Более производительный, чем вложенные LinearLayout/RelativeLayout
- Упрощает создание адаптивных макетов
- Поддерживает сложные анимации и переходы

**Использование в проекте:**
- Используется в XML разметках для создания макетов экранов
- Обеспечивает адаптивность под разные размеры экранов

---

### AndroidX Activity (1.9.0)

**Зависимость:**
```gradle
implementation("androidx.activity:activity:1.9.0")
```

**Назначение:**
- Расширяет функциональность Activity
- Предоставляет дополнительные возможности для работы с Activity
- Поддерживает новые API Android

**Почему выбрана:**
- Часть AndroidX библиотек
- Обеспечивает совместимость с новыми версиями Android
- Необходима для некоторых современных функций

**Использование в проекте:**
- Используется неявно через AppCompatActivity
- Обеспечивает базовую функциональность Activity

## Почему не использованы другие библиотеки

### База данных (SQLite, Room)

**Почему не используется:**
- Для статистики достаточно SharedPreferences
- Нет сложных запросов и связей
- SharedPreferences проще и быстрее для простых данных
- Меньше зависимостей и размера APK

### Сетевые библиотеки (Retrofit, OkHttp)

**Почему не используется:**
- Приложение работает офлайн
- Нет необходимости в сетевых запросах
- Уменьшает размер приложения

### Библиотеки для изображений (Glide, Picasso)

**Почему не используется:**
- Минимальное использование изображений
- Только простые иконки и drawable ресурсы
- Встроенные возможности Android достаточны

### Dependency Injection (Dagger, Hilt)

**Почему не используется:**
- Небольшой размер проекта
- Простая структура зависимостей
- Усложнило бы код без существенной пользы

### Архитектурные компоненты (ViewModel, LiveData)

**Почему не используется:**
- Простое приложение без сложной бизнес-логики
- Нет необходимости в сохранении состояния при поворотах
- Текущая архитектура достаточна для задач

## Версионирование зависимостей

Проект использует централизованное управление версиями через `gradle/libs.versions.toml`:

```toml
[versions]
activity = "1.9.0"
appcompat = "1.6.1"
material = "1.9.0"
constraintlayout = "2.1.4"
junit = "4.13.2"
ext-junit = "1.1.5"
espresso-core = "3.5.1"
```

**Преимущества:**
- Единое место управления версиями
- Легкое обновление зависимостей
- Избежание конфликтов версий

## Итоговая оценка выбора технологий

### Преимущества текущего стека:

1. **Минималистичность** - только необходимые библиотеки
2. **Производительность** - нет лишних зависимостей
3. **Размер APK** - компактное приложение
4. **Совместимость** - поддержка широкого диапазона устройств
5. **Простота поддержки** - понятная структура без излишней сложности

### Возможные улучшения в будущем:

1. **Kotlin** - переход на Kotlin для более современного кода
2. **Jetpack Compose** - современный UI фреймворк
3. **Room Database** - если статистика станет сложнее
4. **Coroutines** - для асинхронных операций
5. **Material Design 3** - обновление до последней версии Material Design
